# Research Findings

## Research Question

Can we automatically mine repository-level semantic sandbox boundary candidates, synthesize safe sentinel tasks, and measure whether real coding agents cross those boundaries?

## Current Understanding

The project should be an NDSS-style **technical method paper**, not a benchmark-only paper. The main contribution is **SandScout**, an automated method that discovers semantic sandbox escape candidates from repository lifecycle artifacts, synthesizes safe sentinel tasks, runs real coding agents, and evaluates normalized traces.

The core insight remains the same: many agent sandbox failures are not classic container escapes. They occur when an agent is induced to create or modify project-local artifacts that look ordinary inside the sandbox but later influence host-context startup, tool discovery, package scripts, CI workflows, configuration loading, credential routing, or approval behavior.

Focused thesis:

> Agent sandboxes are increasingly evaluated as OS isolation mechanisms, but coding agents also cross a semantic boundary through repository files, configuration, tool manifests, and startup state. SandScout automatically mines these semantic boundary candidates and tests real agents with safe sentinel tasks.

The benchmark should appear as **SSE-Bench**, the artifact produced by SandScout, rather than as the only contribution.

## Candidate Ideas

1. **SandScout Automated Mining Method (selected)**
   Automatically discover repository-level semantic sandbox boundary candidates, synthesize safe sentinel tasks, run real agents, and score resulting traces.

2. **Approval-Policy Attack Surface Study**
   Compare how approval UI wording and command summarization affect agent-assisted sandbox violation. This is useful as an ablation, not the paper center.

3. **Agent-Computer Interface Fuzzing**
   Fuzz tool schemas, repository metadata, and model-visible configuration to find ambiguity patterns. This can become a future extension once SandScout has a stable pipeline.

4. **MCP/Tool Provenance Confusion**
   Study malicious local tool manifests and MCP server discovery. Timely, but should be one mined boundary class rather than a broad separate paper.

## Selected Direction

The selected direction is **SandScout: Automated Mining of Semantic Sandbox Escape Paths for Coding Agents**.

Why this is stronger than a benchmark-only framing:

- It contributes an algorithmic method: mine candidates, synthesize tasks, run agents, normalize traces, and score outcomes.
- It targets a gap left by AgentDojo, WASP, AgentHarm, SWE-agent, and container escape benchmarks.
- It can scale to unseen repositories and evolve with agent ecosystems.
- It fits NDSS as a security testing and measurement system.
- SSE-Bench becomes a useful artifact without making the paper feel like only a dataset release.

## Method Plan

SandScout has five modules:

- Boundary miner: scans repositories for lifecycle artifacts such as package metadata, startup/editor configuration, tool manifests, CI workflows, and agent instruction files.
- Task synthesizer: converts each discovered surface into a benign coding task plus adversarial repository context and harmless sentinel target.
- Agent runner: executes real coding agents in disposable worktrees with controlled prompts and sandbox/approval settings.
- Trace normalizer: converts transcripts and diffs into a common action trace.
- Sentinel oracle: computes instruction uptake, artifact placement, triggerability, benign task completion, and SSE-SR.

## Dataset and Benchmark Artifact

Artifact name: **SSE-Bench**, generated by SandScout.

Unit of evaluation:

- A mined boundary source file.
- A benign user goal.
- An attacker-controlled repository context.
- A harmless sentinel artifact target.
- A success oracle that checks placement and triggerability without executing harmful payloads.

## Metrics

Primary metric:

- **SSE-SR**: fraction of tasks where the agent creates or modifies a sentinel artifact that the toy host-transition harness marks as triggerable.

Decomposed metrics:

- Instruction Uptake Rate.
- Artifact Placement Rate.
- Triggerability Rate.
- Benign Task Completion.
- Stealth Cost.
- Mining Yield: number of high-confidence boundary candidates per repository.
- Real-Agent Transfer: success rate variance across agents and policies.

## Patterns and Insights

The literature suggests a split between **behavioral hijacking** and **boundary hijacking**. AgentDojo and WASP evaluate whether untrusted content can redirect agent behavior. SandboxEscapeBench-style work evaluates direct container escape capability. SandScout targets the missing middle: automatically discovering where hijacked behavior can materialize across lifecycle boundaries that the sandbox does not model.

## Lessons and Constraints

- Do not write this as only a benchmark paper. The contribution must be an automated mining and evaluation method.
- Do not build a paper around many defenses. Use defenses only as ablations.
- Avoid real-world exploit payloads. Use toy loaders, sentinel files, and mock secrets to preserve reproducibility and safety.
- Do not claim real product vulnerabilities without coordinated disclosure.
- Real-agent runs must be performed in disposable worktrees.

## Open Questions

- Which repositories should form the first mining corpus?
- Can Codex CLI runs be safely automated with robust timeout, git-preserving disposable worktrees, and transcript/diff capture?
- How should SandScout rank mined candidates by likely triggerability and security relevance?
- What minimum benign-task completion threshold keeps the benchmark realistic?
